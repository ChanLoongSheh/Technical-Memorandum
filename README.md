# 地基红外遥感反演项目 - 技术备忘录库

## 1. 项目概述

本项目旨在通过一个地基红-外探测器系统，对大气垂直温度和露点廓线进行反演。该系统利用不同光谱通道的红外滤光片，测量来自大气的下行辐射，并通过一个复杂的物理前向模型和反演算法，从观测到的电压信号中解算出大气状态。

本项目涉及大气辐射传输（MODTRAN）、仪器物理模型、机器学习、信号处理以及反演理论等多个交叉学科领域。

## 2. 本仓库的目的

本仓库是该研究项目的**官方知识库和思想记录档案**。它的核心目的是以“技术备忘录”（Technical Memorandum）的形式，结构化地记录项目进展中的关键分析、实验设计、理论推导、结果讨论和未来计划。

这样做的好处包括：

*   **思路可追溯**：确保每一个重要结论都有详细的推导和分析过程作为支撑。
*   **知识沉淀**：将零散的思考和讨论固化为结构化文档，形成可复用的知识资产。
*   **促进协作**：为团队成员（包括未来的你）提供一个清晰、统一的信息入口。
*   **阶段性总结**：每一份备忘录都是对一个特定问题或一个阶段性工作的完整总结。

## 3. 如何使用本仓库

### 3.1. 备忘录模板

所有新的技术备忘录都应基于仓库根目录下的 `memo_template.md` 文件进行创建。该模板提供了一个标准化的文档结构，涵盖了从问题定义到结论分析的完整流程，确保了所有备忘录的规范性和可读性。

### 3.2. 创建新的备忘录

1.  复制根目录下的 `memo_template.md` 文件到 `memos/` 目录下。
2.  **重命名**文件，采用统一的命名规范：`YYYY-MM-DD_核心主题关键词.md`。
    *   例如：`2023-10-27_雅可比矩阵鲁棒性分析.md`
3.  根据你当前的研究内容，填写备忘录的各个章节。

## 4. 仓库文件结构

本仓库采用以下结构来组织文件，以实现代码、数据和文档的分离：
```
Technical-Memorandum/
├── .gitignore             # Git忽略文件配置，用于排除大数据、临时文件等
├── README.md              # 仓库主说明文件（即本文档）
├── memo_template.md       # 新备忘录的空白模板
│
├── memos/                   # 存放所有已完成的技术备忘录
│   └── 2023-10-27_雅可比矩阵鲁棒性分析.md
│   └── ...
│
├── src/                     # 存放相关的源代码和脚本
│   ├── analysis/            # 数据分析脚本
│   └── modeling/            # 模型实现代码
│
├── data/                    # 存放原始数据和处理后的数据
│   ├── raw/                 # 原始数据（光谱、廓线等）
│   └── processed/           # 经过预处理的数据
│
└── assets/                  # 存放备忘录中引用的非文本资源
    ├── figures/             # 图片、图表
    └── papers/              # 相关的参考文献PDF
```

```
请遵循此结构，将不同类型的文件放置在对应的目录中，以保持仓库的整洁和有序。
```

---

#### 文件结构建议

我在上面的 `README.md` 文件中已经为你设计了一套清晰、可扩展的文件结构。这里再为你详细解释一下设计的思路：

*   **`memos/`**: 这是最重要的目录，存放你所有的智力成果。将所有备忘录文档放在这里，可以让你快速浏览所有的历史分析。
*   **`src/` (Source)**: 这个目录用来存放你所有的代码。将代码和文档分开是一个非常好的习惯。未来你可能会有数据预处理脚本、模型训练脚本、结果可视化脚本等，都可以放在这里，甚至可以进一步划分子目录（如 `src/analysis`, `src/modeling`）。
*   **`data/`**: 用于存放数据。建议再分子目录为 `raw/` (存放从MODTRAN等原始来源获得的数据，这部分数据原则上不应被修改) 和 `processed/` (存放经过你代码处理后，用于模型训练或分析的数据)。
    *   **特别提醒**: 体积较大的数据文件不适合直接放入Git。你应该配置 `.gitignore` 文件来忽略它们，并考虑使用 Git LFS (Large File Storage) 来管理，或者将数据存放在云存储上，在文档中注明获取方式。
*   **`assets/`**: 用于存放备忘录中引用的“资产”，主要是图片。当你在备忘录中插入分析结果的图表时，把图片文件放在 `assets/figures/` 目录下，然后在Markdown文件中使用相对路径引用 (`../assets/figures/your_figure.png`)。这样可以保持文档的可移植性。
*   **根目录**: 只保留最高层级的配置文件和说明文件，如 `README.md`, `.gitignore`, 和我们的 `memo_template.md`。保持根目录的整洁至关重要。
